City Def — MVP

Питч

Мобильный auto-battler / TD hybrid с коротким ранoм (~5 минут).
Фокус игрока — сборка билда и расстановка до боя, а не контроль во время боя.

В чём фан?

- приятно наблюдать, как отряды автоматически разбирают волны
- интересно собирать билды из случайных офферов
- интересно адаптироваться под противников по лейнам
- интересно развивать состав между битвами (рост числа, опыта, силы)

Основные правила MVP

1) Поле
- 3 лейна.
- 6 слотов игрока:
  - 3 frontline слота (ближний бой, по одному на лейн),
  - 3 backline слота (дальний бой/поддержка, по одному на лейн).
- Противники идут сверху вниз по фиксированным лейнам
- Наши бойцы идут снизу вверх к противникам


2) Структура ранa
- Один ран длится около 3-10 минут.
- Рекомендуемая структура: 3-5 коротких битв.
- Бой полностью автоматический.

3) Фазы каждой битвы
- Preparation:
  - купить новый отряд из случайных предложений,
  - улучшить существующий отряд,
  - переставить отряды по лейнам/рядам.
- Combat:
  - авто-бой без ручных действий.
  - Отряд состоит из нескольких юнитов, но **может иметь только один вражеский отряд в качестве цели атаки** (фокус по одному отряду; при уничтожении цели — автоматическое перенацеливание по правилам боя).
  - В рамках боя с этим отрядом **каждый юнит каждый тик пересчитывает цель атаки** — среди врагов в зоне поражения выбирается **ближайший** вражеский юнит.
- Reward:
  - получить валюту и опыт,
  - выбрать одно действие прогрессии (покупка / апгрейд / экономика).

4) Информация до боя
- Перед стартом показывать иконками, какие враги идут в каждом лейне.
- Показывать ключевые теги волны (например: fast / armored / swarm).
- Ключевое решение игрока: как распределить силу по 3 лейнам.

5) Прогрессия отрядов
- У каждого **юнита** свои характеристики: как минимум **здоровье** и **урон**.
- **После битвы** здоровье выживших юнитов **восполняется до полного** (между битвами в рамках ранa отдельно лечить не нужно).
- Существующие юниты **не улучшаются поштучно** (нет «прокачать одного солдата»): изменить состав силы можно в основном **докупив** юнитов в отряд (или взяв новый отряд из офферов).
- Каждый отряд как целое развивается по осям:
  - численность (от 1 воина до 8) — в т.ч. через докупку юнитов,
  - опыт общий на отряд,
  - качество/сила отряда через апгрейд **на уровне отряда** (не поштучная прокачка отдельных солдат).
- У каждого типа отряда есть понятные преимущества и недостатки
  (оружие, броня, роль).

6) Камень-ножницы-бумага (простая версия)
- Matchup bonus/penalty:
  - выгодный матчап: x1.25
  - нейтральный матчап: x1.00
  - невыгодный матчап: x0.75
- Система должна быть читаемой по иконкам, без сложных таблиц в UI.

7) Разнообразие ранoв
- Основной источник вариативности — случайные офферы между битвами.
- Контроль игрока сохраняется за счёт:
  - лимита слотов,
  - решений по лейнам,
  - выбора между покупкой, апгрейдом и экономикой.

Out of scope для первого MVP

- Знамёна (lane-wide бафы) — добавить в следующей итерации.
- Ручные активные способности во время боя.
- Сложная маршрутизация противников.

Мета-прогрессия

- После победы/поражения выдавать мета-валюту.
- В мете открывать прежде всего новые опции
  (юниты/механики/офферы), а не только постоянные +% к статам.

Критерии успеха MVP

- Игрок понимает смысл лейновых решений за первый ран.
- Раны ощущаются разными за счёт рандома офферов.
- Контры заметны и полезны на практике.
- Наблюдать авто-бой интересно даже без ручного контроля.
