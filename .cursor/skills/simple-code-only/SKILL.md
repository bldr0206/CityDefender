# Simple Code Only

## Принцип

Пиши код минималистично и просто: джун должен понять его без комментариев.

Лучший и красивый код - код, где мало строчек и все работает.

## Как писать

- Выбирай самое прямое решение, которое закрывает задачу.
- Удаляй лишние промежуточные переменные, методы и классы, если они не повышают читаемость.
- Не добавляй абстракции "на будущее".
- Не делай safe-проверки без надобности.
- Не покрывай код защитой от состояний, которые невозможны по текущим инвариантам.
- Если код требует комментария, сначала попробуй упростить сам код.
- Оставляй комментарии только там, где они объясняют важную причину, а не пересказывают строки кода.

## Перед завершением

Проверь отредактированный фрагмент:

- Можно ли сделать короче без потери ясности?
- Можно ли назвать переменные проще?
- Есть ли проверки, которые ничего реально не защищают?
- Есть ли дублирование, которое убирается без новой сложной абстракции?
